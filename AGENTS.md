# AGENTS.md — kinesis-mock

A mock of the AWS Kinesis API for local testing, written in Scala 3 (Typelevel/cats-effect stack). Cross-built for the JVM (fat JAR) and Scala.js (Node `main.js`, published to npm as `kinesis-local` and as a Docker image `ghcr.io/etspaceman/kinesis-mock`). Main class: `kinesis.mock.KinesisMockService`. Serves HTTPS on 4567 and HTTP on 4568.

## Toolchain

- **sbt** (version in `project/build.properties`); **Scala 3** (version in `build.sbt`). Tool/lib versions are Scala-Steward-managed — read them from the build files, don't trust any number quoted here.
- Built on sbt-typelevel; the build is a `tlCrossRootProject` aggregating `kinesis-mock-jvm` and `kinesis-mock-js`. CI calls projects `kinesis-mock-rootJVM` / `kinesis-mock-rootJS`.
- Dependencies live in `project/LibraryDependencies.scala`. Stack: cats / cats-effect, http4s (ember-server, dsl, circe), fs2, circe, borer (CBOR), ciris (config), enumeratum, log4cats, scodec-bits. Tests: munit + munit-cats-effect + scalacheck(-effect); AWS SDK v2 / KPL / KCL are JVM `Test`-only.

The pre-commit test/build path is **whatever CI runs**. CI is generated by sbt-typelevel from the build, so `.github/workflows/ci.yml` is the source of truth — if it changes on a working branch, update the commands below to match (and never hand-edit the generated files; see Gotchas). CI matrix: **Scala 3** × projects `kinesis-mock-rootJVM` and `kinesis-mock-rootJS`, JDK 21.

### Validate like CI (run before committing)

Run for **each** project — `kinesis-mock-rootJVM` and `kinesis-mock-rootJS`:

```
sbt 'project kinesis-mock-rootJVM' '++ 3' test
sbt 'project kinesis-mock-rootJVM' '++ 3' headerCheckAll scalafmtCheckAll 'project /' scalafmtSbtCheck
sbt 'project kinesis-mock-rootJVM' '++ 3' 'scalafixAll --check'
```

The JS project additionally link-checks (CI steps `scalaJSLink` before tests, `Link JS` after):

```
sbt 'project kinesis-mock-rootJS' '++ 3' Test/scalaJSLinkerResult
sbt 'project kinesis-mock-rootJS' '++ 3' fastLinkJS
```

Integration tests (Docker-based — see Gotchas): bring the JS-image compose stack up, run `itTest` on the JVM, tear down:

```
sbt 'project kinesis-mock-rootJS' '++ 3' dockerComposeUp
sbt 'project kinesis-mock-rootJVM' itTest
sbt 'project kinesis-mock-rootJS' '++ 3' dockerComposeDown
```

These are **heavy** — run them at the **end of an implementation** (e.g. after executing a plan), not on every commit. The exception is when your changes touch the integration tests or the docker environment themselves, in which case run them as part of validating those changes.

### Convenience aliases (local shortcuts — not required for CI validation)

- `sbt pretty` — auto-fix headers + scalafix + scalafmt (and regen workflows); `prettyCheck` is the check-only form. Run `pretty` locally to fix whatever the CI format/lint checks above would flag.
- `sbt 'project kinesis-mock-rootJVM' assembly` — JVM fat JAR → `docker/image/lib/kinesis-mock.jar`.
- `sbt 'project kinesis-mock-rootJS' buildDockerImage` — JS-based Docker image (buildx; multi-arch on push).
- `sbt 'project kinesis-mock-rootJS' Compile/npmPackage` — build the npm `kinesis-local` package.

## Architecture / source layout

Single crossProject (`CrossType.Pure`) rooted at `kinesis-mock/`. Note the non-default source dirs (configured in `build.sbt`):

- `kinesis-mock/src/main/scala/kinesis/mock/` — shared business logic for both platforms. Entry point `KinesisMockService.scala`; routing `KinesisMockRoutes.scala`; config `KinesisMockServiceConfig.scala`. Key dirs:
  - `api/` — one file per Kinesis API action, each a `*Request`/`*Response` case class (CreateStreamRequest, PutRecordsRequest, SubscribeToShardRequest, …) whose request methods mutate `Ref[IO, Streams]` and return `IO[Response[…]]`; go here to change how an operation behaves or its wire shape.
  - `cache/` — the in-memory engine and lifecycle: `Cache` (top-level state holder wrapping `Ref[IO, Streams]`, dispatches API calls, persistence), `CacheConfig`/`PersistConfig`, `CacheSemaphores`, and `SubscriptionRegistry` (one-active-SubscribeToShard-per-shard tracking); go here for state orchestration, locking, config, or persistence.
  - `models/` — the domain data types and newtypes (Shard, StreamData, Streams, Consumer, SequenceNumber, StreamArn, ShardId, enums like StreamStatus/EncryptionType); go here to change data structures, ARN/ID formats, or stream/shard representation.
  - `validations/` — the single `CommonValidations` object with reusable validators (stream/ARN resolution, name/limit/shard existence checks) returning `Response[…]`; go here for shared request-validation rules and error semantics.
  - `instances/` — typeclass-instance modules per library: `circe` (JSON/CBOR Instant codecs), `borer` (CBOR<->circe bridge), `ciris` (config decoders), `http4s` (Entity en/decoders, content-type negotiation); go here for serialization, config-parsing, or HTTP-body codec behavior.
  - `syntax/` — small Scala extension-method helpers: `either` (`sequenceWithDefault`) and `semaphore` (`tryAcquireRelease`); go here for shared syntactic combinators.
  - `eventstream/` — AWS event-stream (vended-stream) binary framing for SubscribeToShard: `EventStreamMessage` (prelude/headers/payload/CRC wire encoding), `EventStreamHeader`, `Crc32`; go here for the SubscribeToShard streaming wire format.
  - `retry/` — a vendored cats-retry port (RetryPolicies, RetryPolicy, RetryDetails, RetryStatus, Fibonacci backoff, Sleep) providing `retryingOnFailures`/`OnErrors` combinators; go here for retry/backoff logic.
- `kinesis-mock/src/main/scalajvm/` — JVM-only impls (`AES.scala`, `TLS.scala`).
- `kinesis-mock/src/main/scalajs/` — JS-only impls (`AES.scala`, `TLS.scala`, same API).
- `kinesis-mock/src/main/resources/` — `server.json` (JS self-signed cert), `server.jks` (JVM keystore), `logback.xml`.
- `kinesis-mock/src/test/{scala,scalajvm}/` — shared tests + JVM-only tests.

> **Maintenance:** keep the key-dirs list above in sync — when a `kinesis/mock/` subpackage is added or removed, update these bullets.

Build plumbing in `project/`: `KinesisMockPlugin.scala` (command aliases, CI workflow gen, Scala version), `DockerImagePlugin.scala` (`buildDockerImage`/`pushDockerImage` via buildx), `DockerComposePlugin.scala` (`dockerComposeUp/Down/Logs/Ps`), `ScalacSettings.scala`. Docker bits in `docker/`: `Dockerfile` (Node alpine base, runs `main.js`), `Dockerfile.JVM` (Zulu JDK base, runs the JAR), `docker-compose.yml` (mock + localstack + dynamodb-local + awscli for functional tests).

## Runtime config (env vars)

Configured via ciris env vars. Common: `INITIALIZE_STREAMS` (`name:shards:region` comma list, default 4 shards), `KINESIS_MOCK_TLS_PORT` (4567), `KINESIS_MOCK_PLAIN_PORT` (4568), `LOG_LEVEL`/`ROOT_LOG_LEVEL`, `SHOULD_PERSIST_DATA`+`PERSIST_PATH`, `AWS_ACCOUNT_ID`/`AWS_REGION`. CI also toggles `CBOR_ENABLED` and `SERVICE_PORT`. Full table in `README.md`.

## Gotchas

- **Cross build**: always specify `project kinesis-mock-rootJVM` or `...rootJS`; bare `test` is ambiguous. The Docker image and npm package are built from the **JS** linker output, not the JVM JAR.
- `Test / parallelExecution := false` on the JVM — JVM tests run serially by design; don't "fix" this.
- **Integration tests** (`itTest`, MUnit tag `integration`) require `dockerComposeUp` running first (localstack needs `LOCALSTACK_AUTH_TOKEN`); they run on the JVM project against the JS Docker image.
- The fat JAR's assembly output path and the JS linker output path are both hard-pinned to `docker/image/lib/` — that dir is the Docker build context input.
- **Generated files — never hand-edit**: `.github/workflows/*.yml` (sbt-typelevel, `sbt githubWorkflowGenerate`) and `.mergify.yml` (sbt-typelevel-mergify, `sbt mergifyGenerate`). Change the build settings and regenerate; `sbt pretty` regenerates both.
- Scalafmt reformat commits are listed in `.git-blame-ignore-revs` — keep that updated when bumping scalafmt.

## Code style

Functional Scala 3 on the Typelevel ecosystem. scalafmt (`.scalafmt.conf`, scala3 dialect, new-syntax rewrites, custom import groups) and scalafix (`.scalafix.conf` `DisableSyntax`: no `var`, `null`, `throw`, `return`, `while`, `asInstanceOf`, default args, etc.) are enforced in CI. Compilation is `-Xfatal-warnings`, so warnings break the build. Run `sbt pretty` before committing.
